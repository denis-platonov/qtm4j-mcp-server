- [ ] **Step 2: Update the publish section to emphasize the tag-driven release flow**

```md
### Publish

This repository uses a tag-driven GitHub Actions release workflow.

1. Align `package.json` and `server.json` to the release version.
2. Run local verification:

```bash
npm run build
npm run test:run
```

3. Commit the release-prep changes.
4. Create and push the release tag:

```bash
git tag v1.1.0
git push origin sync/desktop-qtm4j-source
git push origin v1.1.0
```

5. GitHub Actions will verify the tag matches both manifests, publish to npm, and publish `server.json` to the MCP Registry.
